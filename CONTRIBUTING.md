# Contributing to TARL-Net

Thank you for your interest in contributing to **TARL-Net (Threat-Aware Adaptive Representation Learning Network)**.

This project is a PhD-level research implementation in the field of **Cybersecurity, Deep Learning, and Intrusion Detection Systems (IDS)**. Contributions are welcome, but must follow academic and technical standards.

---

## Important Notice

This repository is primarily a **research prototype**.

- It is not designed for production deployment.
- Contributions should prioritize scientific validity, reproducibility, and clarity.
- All modifications should maintain consistency with the research objectives (Zero-Day detection, OSR, SSL, XAI, Continual Learning).

---

## How to Contribute

You can contribute in the following ways:

### 1. Bug Reports
Please open an issue with:
- Clear description
- Steps to reproduce
- Expected vs actual behavior
- Environment details

---

### 2. Feature Suggestions
Feature proposals should align with:

- Cybersecurity research relevance
- Machine learning rigor
- Reproducibility
- Model interpretability or robustness

---

### 3. Code Contributions

Before submitting a pull request:

- Ensure code follows PEP8 standards
- Keep notebook structure clean and reproducible
- Avoid unnecessary complexity
- Provide comments only when necessary for scientific clarity
- Test changes on at least one dataset pipeline

---

## Pull Request Guidelines

All pull requests must include:

- Clear description of changes
- Motivation and research justification
- Experimental validation (if applicable)
- Impact on existing results

---

## Scientific Integrity

Because this is an academic project:

- Do NOT introduce unverifiable claims
- Do NOT modify evaluation protocols without justification
- Do NOT optimize results in a way that harms reproducibility

---

## Code of Conduct

By participating, you agree to follow the `CODE_OF_CONDUCT.md`.
